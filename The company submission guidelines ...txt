Step-by-Step Guide: Create React App and Push to GitHub
1. Create a New React App
1. Install Node.js (LTS) from https://nodejs.org/
2. Open your terminal and run the following commands:

npx create-react-app my-app
cd my-app
code .


2. Set Up Git (First Time Only)
If you haven’t configured Git globally yet, run these commands:

git config --global user.name "Your Name"
git config --global user.email "you@example.com"


3. Initialize Git in Your Project
1. Initialize the Git repository:
   git init
2. Ensure `.gitignore` includes `node_modules/` (it’s already added by Create React App).
3. Stage and commit your initial code:

git add .
git commit -m "Initial commit: Create React App"


4. Create a Repository on GitHub
1. Go to GitHub and click **New Repository**.
2. Name your repository (e.g., `my-app`) and keep it **empty** (no README, .gitignore, or license).
5. Connect Local Project to GitHub
Replace USER and REPO with your GitHub username and repository name:

git branch -M main
git remote add origin https://github.com/USER/REPO.git
git push -u origin main


6. Future Updates (Making Changes)
Whenever you make changes to your code, use:

git add -A
git commit -m "Describe your change"
git push




Deployment Guide: Vercel and Netlify
Option 1: Vercel (Recommended for React, Vite, or Next.js)
1. Go to https://vercel.com and log in.
2. Click 'New Project' → 'Import Git Repository' → select your GitHub repository.
3. Framework preset will be auto-detected; if asked, choose the correct one (React, Vite, or Next.js).
4. Build command (defaults are fine):
   - Create React App (CRA): npm run build
   - Vite/Next: auto
5. Output directory:
   - CRA: build
   - Vite: dist
   - Next: .vercel/output (auto)
6. Add Environment Variables if needed.
7. Click 'Deploy'.
8. (Optional) Add custom domain under Vercel → Domains.
9. Every git push to the repository will auto-deploy.

Option 2: Netlify
1. Go to https://app.netlify.com and log in.
2. Click 'Add new site' → 'Import an existing project'.
3. Connect GitHub and select your repository.
4. Build settings:
   - Build command: npm run build
   - Publish directory:
     - CRA: build
     - Vite: dist
5. Add Environment Variables if needed.
6. Click 'Deploy site'.
7. (Optional) Go to 'Domain settings' to add a custom domain.
8. Every git push to the repository will auto-deploy.

Front-End Assessment Guidelines
Assessment Nature
This is an Open Book Assessment. Candidates may refer to documentation, learning resources, and online references while attempting the assessment.
Important Note on AI Usage
While AI tools may be available, this assessment is designed to evaluate the candidate's understanding of Front-End Development concepts, problem-solving ability, and coding skills. Excessive dependency on AI-generated solutions may negatively impact the evaluation process.
Evaluation Criteria
Candidates will be evaluated on:
Understanding of requirements and problem-solving approach.
Ability to explain the implemented solution.
Code quality, readability, and maintainability.
Proper component structure and state management.
Handling of edge cases and user interactions.
Knowledge demonstrated during the technical interview.
Post-Assessment Verification
Shortlisted candidates will undergo a technical discussion where they may be asked to:
Explain their implementation approach.
Modify parts of their submitted solution.
Debug issues in their own code.
Add new features to the existing application.
Answer questions related to the concepts used in their submission.
Candidates who are unable to explain or modify their submitted code may be disqualified from the selection process.
Recommended Practices
Write the solution in your own style.
Use meaningful variable and function names.
Build the application incrementally rather than relying on complete generated solutions.
Ensure you understand every line of code submitted.
Integrity Declaration
By submitting this assessment, I confirm that I understand the implementation provided in my submission and can explain, debug, and extend the solution during the interview process if required.


NOTE: Submissions using AI completely to solve will be considered void 

Submission Instructions
Step 1: Prepare Your Project
After completing the assessment, ensure that your project is working correctly and contains all the required source code files.
Before creating the ZIP file:
Remove unnecessary files and folders.
Do not include large dependency folders such as node_modules.
Ensure that all source code files required to run the project are included.
Verify that your project structure is organized and complete.
Step 2: Create a ZIP File
Compress your project folder into a single ZIP file using the following naming convention:
FullName_AssessmentSubmission.zip
Example:
JohnDoe_AssessmentSubmission.zip
Step 3: Upload to Google Drive
Open Google Drive.
Upload the ZIP file to your Google Drive.
After the upload is complete, right-click on the ZIP file and select Share.
Under General Access, change the permission to Anyone with the link.
Ensure the access level is set to Viewer.
Copy the shareable link.
Step 4: Submit the Link
Submit the copied Google Drive link through the assessment submission form or the provided submission portal.
Important Security and Privacy Notice
Before uploading your ZIP file, carefully review your project and remove any confidential or personal information.
Do NOT include:
Personal identification documents.
Passwords or credentials.
API keys, access tokens, or secret keys.
Environment files (.env) containing sensitive information.
Database credentials.
Private certificates or authentication files.
Personal photos, documents, or unrelated files.
Candidates are solely responsible for ensuring that no confidential, sensitive, or personal information is included in the submitted ZIP file.
Verification
Please verify the following before submission:
The ZIP file opens successfully.
The Google Drive link is publicly accessible.
The project source code is complete.
No sensitive information is included.
The shared link does not require additional access requests.
Failure to provide a working and accessible submission link may result in the submission not being evaluated.
